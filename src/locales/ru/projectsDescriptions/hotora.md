# Inputs

**Inputs** - это лёгкая и расширяемая библиотека для JavaScript и TypeScript, предназначенная для обработки пользовательского ввода. Она позволяет описывать горячие клавиши, последовательности действий, жесты и любые пользовательские комбинации через единый API, независимо от источника событий.

Библиотека была разработана как универсальное решение для сложных веб-приложений, где требуется гораздо больше возможностей, чем предоставляет обычная обработка `keydown` и `keyup`. В отличие от большинства существующих решений, Inputs умеет работать с иерархией областей (Scopes), поддерживает последовательности действий, безопасно используется в SSR-приложениях и позволяет подключать собственные источники событий.

---

## Основные возможности

- ⌨️ Регистрация одиночных клавиш, комбинаций и последовательностей
- 🎯 Локальные и глобальные области обработки (Scopes)
- 🌳 Наследование областей через DOM-дерево
- 🚫 Управление распространением событий (`stopPropagation`)
- 👆 Автоматическое определение активного элемента
- 👁️ Работа только с видимыми элементами
- 🔌 Поддержка собственных EventProvider
- ⚡ SSR-friendly архитектура с ленивой инициализацией
- 🧩 Полностью типизированный API

# Проблема

Во многих проектах обработка пользовательского ввода быстро превращается в большое количество разрозненных обработчиков:

- десятки `keydown`;
- постоянные проверки активного элемента;
- ручное отключение обработчиков;
- конфликты между модальными окнами, редакторами и глобальными горячими клавишами.

Особенно заметно это становится в крупных приложениях: редакторах, административных панелях, графических редакторах, игровых интерфейсах и сложных SPA.

Inputs решает эту проблему, предоставляя единый менеджер пользовательского ввода.

---

# Архитектура

В основе библиотеки лежит объект **InputsManager**.

Он отвечает за:

- регистрацию обработчиков;
- отслеживание активных комбинаций;
- выполнение последовательностей;
- определение активной области;
- распространение событий между Scope;
- взаимодействие с EventProvider.

При этом сама библиотека никак не зависит от браузера.

Работа с DOM реализована через отдельный `DOMKeyboardEventProvider`, благодаря чему архитектура остаётся полностью расширяемой.

Это позволяет использовать Inputs не только в браузере, но и в SSR, Canvas-приложениях, игровых движках или любых собственных окружениях.

---

# Scoped Architecture

Одной из главных особенностей библиотеки является система **Scopes**.

Каждый обработчик может быть привязан к определённому элементу и собственной области.

Во время возникновения события библиотека автоматически:

1. определяет активный элемент;
2. строит цепочку родителей;
3. собирает соответствующие Scope;
4. вызывает обработчики от локального Scope к глобальному.

Такой подход позволяет создавать независимые части интерфейса, не опасаясь конфликтов между ними.

Например:

- редактор текста;
- модальное окно;
- контекстное меню;
- глобальные горячие клавиши.

Все они могут использовать одинаковые сочетания клавиш, не влияя друг на друга.

# Provider-based архитектура

Inputs полностью построен вокруг абстракции **EventProvider**.

Библиотека ничего не знает о том, откуда поступают события.

Источник может быть любым:

- браузер;
- Canvas;
- WebGL;
- игровой движок;
- тестовое окружение;
- серверный рендеринг.

Для этого достаточно реализовать интерфейс `EventProvider`.

Благодаря этому библиотека легко интегрируется практически в любую среду выполнения.

---

# SSR и Lazy Provider

Обычные библиотеки горячих клавиш часто невозможно безопасно использовать во время серверного рендеринга.

Inputs решает эту проблему с помощью `LazyEventProvider`.

Он откладывает создание настоящего EventProvider до момента, когда среда готова к работе (например, после появления объекта `window`).

Это позволяет использовать библиотеку в:

- Next.js;
- Nuxt;
- Astro;
- Remix;
- любых SSR-приложениях.
